coldGUI is a graphical user interface toolkit powered by Pygame

Altough coldGUI uses Pygame, the main purpose of it is not being a library to used while making games.
It provides various themeable widgets and a simple interface to let users achieve their goals easier.



You can see a cool little example in the right ;)
You can also find more examples in the Examples section.
coldGUI is open-source, licensed under GNU GPL v3. You can contribute if you will, the source is hosted on github link here

a) Introduction

A simple coldGUI project's work flow should be like this:
importing coldGUI > creating the top level window > adding widgets > starting the main loop

from coldgui import * #Import coldGUI to your project

mainwindow = MainWindow((800, 600), title="My project!") #Create your top level window

mainwindow.start() #Start the main loop

b) Basics of Widgets

A widget is simply an interactable component to control various events.
They may not be inceractable always, there are different types of widgets to fulfil any task.
Also there are a few themes for the widgets, win95 style being the default one.




1) Label

Label is simply a one-line text piece drawn on screen.

A generic win95 styled label


You can create the example label above like this:


my_label = Label(mainwindow, position=(30, 30), text="Label", font-size=18)

As you can see, there are some arguments passed in while creating a Label, every widget needs these parameters to customize itself. The first argument must be entered because it's our widget's parent (the main window in this case).

Here are some important ones of these arguments.

Argument Description
parent Widget's parent. Mostly the main window but it could also be a frame
position Widget's position relative to it's parent
text Widget's text content
font_size Size of the font used in drawing the text
... For other fields and more detailed explanation, see API Reference



2) Button

Sometimes you need to click something to de-stress, don't you? This widget is created for you.

A generic win95 styled button


You can create the example button above like this:


my_button = Button(mainwindow, position=(30, 30), text="Button", font-size=18)

There are similar arguments as Label's. And here are some important ones

Argument Description
parent Widget's parent. Mostly the main window but it could also be a frame
position Widget's position relative to it's parent
text Widget's text content
font_size Size of the font used in drawing the text
width Forces button to size itself to specific width
height Forces button to size itself to specific height
padding Space between widget's border and widget's content
... For other fields and more detailed explanation, see API Reference

Now, you see three new arguments named width, height and padding. As their descriptions tells, they used to customize the button.
let's start with padding.

Four buttons with different padding sizes

Widget's padding size determines the space between it's border and it's content.

By default, widget's size is calculated relative to it's font size, border size and padding size (see API Reference for further details.) but user can force the widget to a specific size.

Here you can see we have four buttons with different padding sizes, but they have fixed width; because of this, the padding size only affects their height.



3) TextBox

Textbox is a widget to let the user input text.

A generic win95 styled textbox


You can create the example textbox above like this:


my_textbox = TextBox(mainwindow, position=(30, 30), placeholder="TextBox", font-size=18)

Some of the important arguments listed below

Argument Description
parent Widget's parent. Mostly the main window but it could also be a frame
position Widget's position relative to it's parent
placeholder A placeholder text to show widget's task
font_size Size of the font used in drawing the text
... For other fields and more detailed explanation, see API Reference

Textboxes doesn't have an argument specificly for the text content of the widget unlike the others, because the user is expected to register that.
Instead, there is a placeholder argument which is shown when the widget is not in use, you can think it as a hint text.
Notice: Textboxes also have width, height and padding arguments like buttons. To see all arguments and further detail, see API Reference
